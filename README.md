`git init`
initializes a new Git repository in the current folder so Git can start tracking the project.

`git clone <repo_url>`
creates a copy of an existing GitHub repository on your local machine.

`git status` 
shows the current state of your project files, including modified or untracked files.

`git add <file>` 
adds a specific file to the staging area so it can be included in the next commit.

`git add .` 
adds all changed files in the current directory to the staging area.

`git commit -m "message"` 
saves the staged changes to the repository with a descriptive message.

`git push origin main` 
uploads your local commits to the main branch of the remote GitHub repository.

`git pull origin main` 
downloads and merges the latest changes from the remote repository to your local project.

`git fetch` 
downloads updates from the remote repository but does not merge them automatically.

`git branch` 
displays a list of all branches in the repository.

`git branch <branch_name>` 
creates a new branch for developing features separately.

`git checkout <branch_name>` 
switches your working directory to the specified branch.

`git checkout -b <branch_name>` 
creates a new branch and switches to it at the same time.

`git merge <branch_name>` 
combines the changes from another branch into the current branch.

`git log` 
shows the history of commits made in the repository.
`git remote -v` 
displays the remote repositories connected to your project.
